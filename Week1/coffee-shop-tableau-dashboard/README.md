# ☕ Coffee Shop Sales Dashboard – Tableau (Week 1)

This project is part of my Tableau learning journey. In this week's practice, I worked on building a sales dashboard in Tableau using a coffee shop transactional dataset. The purpose of this project was to understand how raw sales data can be transformed into useful business insights that support managerial strategy and decision-making.

## 📊 Dashboard Preview

![Coffee Shop Dashboard Screenshot](dashboard/tableau%20week1%20image.png)

## 🎯 Business Value & Objectives

The main objective of this dashboard was to create visualizations that help answer four important business questions:
1. **How is revenue changing over time?**
2. **Which product categories generate the most revenue?**
3. **Which products are performing best?**
4. **Which products are underperforming?**

These insights can support decisions related to inventory planning, menu strategy, promotions, pricing, and overall business performance monitoring.

## 📁 Dataset Overview

The dataset used in this project contains coffee shop transaction records. Each row represents a single product purchase made during a transaction. 

Key columns used in this project include:
*   `transaction_date`: Used to analyze revenue over time.
*   `product_category`: Used to compare categories such as coffee, tea, bakery, and drinking chocolate.
*   `product_detail`: Used to identify individual products.
*   `subtotal`: Used as the primary revenue metric across all charts, representing the value generated from each transaction.

Understanding and configuring the correct data types in Tableau (e.g., date for `transaction_date` and numeric for `subtotal`) was a crucial first step before building any visualizations.

## 🛠️ Step-by-Step Tutorial: How I Built the Visualizations

The workflow started with importing the CSV dataset into Tableau Desktop and verifying field interpretations (e.g., ensuring date fields were properly parsed). I created individual worksheets for each required graph before assembling the final dashboard. Here is exactly how I built each one:

### 1️⃣ Monthly Revenue (Line Chart)
*This chart reveals seasonality, trends, and sales peaks over time.*

**Steps:**
1. Drag **`transaction_date`** to the *Columns* shelf.
2. Change the date level to **`MONTH(transaction_date)`** to view grouped trends.
3. Drag **`subtotal`** to the *Rows* shelf.
4. Ensure the measure is set to **`SUM(subtotal)`**.
5. Change the *Marks* type to **Line**.
6. Format the line and tooltips to display revenue figures cleanly.

### 2️⃣ Revenue by Product Category (Horizontal Bar Chart)
*This visualization immediately highlights the strongest-performing product groups.*

**Steps:**
1. Drag **`product_category`** to the *Rows* shelf.
2. Drag **`subtotal`** to the *Columns* shelf.
3. Ensure the aggregation is **`SUM(subtotal)`**.
4. Change the *Marks* type to **Bar**.
5. Sort the bars in **descending order**.
6. Drag **`subtotal`** to the *Label* shelf so the revenue generated by each category is easily readable.

### 3️⃣ Top 5 Products (Filtered Bar Chart)
*Identifies the best-selling individual products driving the business.*

**Steps:**
1. Drag **`product_detail`** to the *Rows* shelf.
2. Drag **`subtotal`** to the *Columns* shelf.
3. Ensure the measure is **`SUM(subtotal)`**.
4. Apply a filter on **`product_detail`**:
   * Select **Top**.
   * Choose **Top 5 by SUM(subtotal)**.
5. Sort in **descending order**.
6. Set the bar color to a **positive/primary color** (e.g., blue or green) to signify high performance.
7. Include **`subtotal`** labels on the bars.

### 4️⃣ Bottom 5 Products (Filtered Bar Chart)
*Highlights underperforming products that may need promotion, pricing review, or removal.*

**Steps:**
1. Duplicate the *Top 5 Products* worksheet.
2. Edit the filter on **`product_detail`**:
   * Change from *Top* to **Bottom**.
   * Choose **Bottom 5 by SUM(subtotal)**.
3. Sort in **ascending order** so the absolute worst performer is at the top.
4. Set the bar color to a **warning color** (e.g., red or orange) to visually indicate underperformance.

### 5️⃣ Dashboard Assembly & Design
*Combined all worksheets into a clean, unified business layout.*

**Steps:**
1. Create a new dashboard and set a fixed, standard layout size.
2. Add a strong title explaining the dashboard's purpose.
3. Use **Dashboard Containers** to arrange the charts logically:
   * *Top Section:* Monthly Revenue Line Chart.
   * *Bottom Left:* Revenue by Category.
   * *Bottom Right:* Top 5 & Bottom 5 Products arranged vertically.
4. Ensure the color scheme is consistent, making the insights visually intuitive for non-technical users.
